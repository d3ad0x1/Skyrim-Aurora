# Skyrim: Aurora Style Guide

> Design principles and mod selection guidelines.

---

# Philosophy

Aurora is not designed to change Skyrim.

Aurora is designed to improve Skyrim.

The player should immediately recognize the original game.

---

# Core Values

## Vanilla+

Always preserve Skyrim's original identity.

---

## Performance First

FPS is more important than visual effects.

Every graphical improvement must justify its performance cost.

---

## Quality over Quantity

Aurora does not aim to include the highest number of mods.

Aurora aims to include the best mods.

---

## Stability

A stable game is more important than adding one more feature.

---

## Compatibility

Mods should require as few compatibility patches as possible.

---

## Lore Friendly

Everything should feel like Bethesda could have created it.

# Mod Acceptance Checklist

Before adding any mod:

- Does it fit Vanilla+?
- Is it lore-friendly?
- Is it actively maintained?
- Is it compatible with AE?
- Does it reduce FPS significantly?
- Does it require many patches?
- Does it duplicate another mod?
- Is there a better alternative?

If two or more answers are negative, the mod should not be included.
