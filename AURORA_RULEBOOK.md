# Skyrim: Aurora Rulebook

> Official development rules for Skyrim: Aurora

Version: 1.0

---

# Mission

Aurora exists to modernize Skyrim without changing its identity.

Every decision made during development must support this goal.

---

# The Golden Rules

## Rule #1

Vanilla First.

If a feature changes Skyrim more than it improves Skyrim,
it does not belong in Aurora.

---

## Rule #2

Performance First.

Visual quality must never come at the cost of playability.

Aurora should remain enjoyable on mid-range hardware.

---

## Rule #3

Quality over Quantity.

Adding more mods does not make a better modlist.

Every mod must earn its place.

---

## Rule #4

Stability Above Everything.

A stable game is always more important than one additional feature.

---

## Rule #5

No Feature Creep.

Aurora is not trying to become:

- Nolvus
- Requiem
- Wildlander
- Living Skyrim

Aurora follows its own vision.

---

# Mod Acceptance Rules

Every new mod must answer YES to the following questions.

## Compatibility

- Compatible with Anniversary Edition
- Works with latest SKSE
- Compatible with current Aurora version

---

## Maintenance

- Still maintained
- No abandoned projects unless absolutely necessary

---

## Stability

- No critical bugs
- No save corruption
- No script spam

---

## Performance

- Acceptable FPS impact
- Acceptable VRAM usage
- Fast loading times

---

## Community

- Good reputation
- Positive feedback
- Mature project

---

## Purpose

Every mod must have a clear purpose.

Mods added "because they look cool" are not accepted.

---

# Reasons for Rejection

Aurora will reject mods that:

- Break vanilla quests
- Require dozens of patches
- Are abandoned
- Cause frequent crashes
- Introduce heavy scripting
- Duplicate existing functionality
- Significantly reduce FPS
- Conflict with Aurora philosophy

---

# Gameplay Rules

Aurora will NOT include:

- Souls-like combat
- Hardcore survival
- Complete perk overhauls
- Complete magic overhauls
- Economy overhauls
- Level scaling removal
- NSFW content
- Anime content
- Meme mods
- Joke mods

---

# Graphics Rules

Allowed

- HD textures
- Better meshes
- Better lighting
- Better weather
- Better water
- Better vegetation

Not Allowed

- Plastic faces
- Cartoon visuals
- Unrealistic colors
- Over-saturated ENBs

---

# Character Rules

Characters should look realistic.

Not like fashion models.

NPCs should remain believable citizens of Skyrim.

---

# Quest Rules

Preferred

- Lore Friendly
- Fully voiced
- Stable
- Well written

Rejected

- Poor writing
- Broken quests
- Unfinished projects
- AI-generated voice acting (unless exceptionally high quality)

---

# Performance Budget

Aurora targets:

1080p

60 FPS

Mid-range PC

SSD

No ENB required

Community Shaders by default

---

# Optional Content

Heavy visual improvements belong in optional packs.

Never in Aurora Core.

Examples:

- 4K Textures
- ENB
- Body Physics
- Dense Flora
- Cinematic Effects

---

# Aurora Core Principles

Aurora Core must always be:

- Stable
- Fast
- Lightweight
- Modular
- Maintainable

---

# Code of Development

When adding a new mod:

1. Read the documentation.
2. Check compatibility.
3. Install only one mod.
4. Launch the game.
5. Test.
6. Benchmark FPS.
7. Check logs.
8. Commit changes.
9. Update documentation.

Never install multiple unknown mods at once.

---

# Testing Policy

Every release must pass:

- New Game Test
- Existing Save Test
- City Performance Test
- Combat Test
- Quest Test
- Crash Test
